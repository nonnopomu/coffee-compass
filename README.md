## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

Coffee Compass Club のアイデアは、カフェを探す中で感じた  
**「品質にこだわったコーヒーを提供するカフェを見つけにくい」**  
という経験から生まれました。

地図アプリや口コミサイトでは、場所・営業時間・雰囲気・写真・評価などは簡単に確認できます。  
一方で、コーヒーに興味を持ち始めた頃は、どのカフェがコーヒーに力を入れているのかを判断するのが難しいと感じました。

また、コーヒーに力を入れているカフェを見つけても、複数の豆やメニューの中から自分の好みに合う一杯を選ぶのは簡単ではありませんでした。  
産地・焙煎度・テイストノートを見ても、それが自分の好みとどう結びつくのかが分かりにくく、せっかくカフェに行っても選びきれない感覚がありました。

この体験から、まずは **品質にこだわったコーヒーを提供するカフェを見つけやすくすること**、そのうえで **自分に合う一杯を選びやすくすること** を支援したいと考えました。

---

### 2-2. なぜそれが「気になった」のか

コーヒーの楽しさは、詳しい人だけのものではなく、  
**良いお店や自分に合う一杯と出会うことで、誰でも少しずつ広げていけるもの**  
だと感じているからです。

コーヒーには、産地・焙煎度・精製方法・抽出方法・テイストノートなど多くの要素があります。  
しかし、最初からそれらを理解し、店選びや豆選びに活かすのはハードルが高いと感じました。

自分自身も、最初は口コミや雰囲気を頼りにお店を探していましたが、飲む経験が増えるにつれて、  
「浅煎りが好きかもしれない」  
「フルーティーな味が好みかもしれない」  
「深煎りのコクがあるコーヒーを飲みたい気分かもしれない」  
といった自分なりの好みを意識するようになりました。

そのため、単にカフェを一覧で探すだけではなく、飲んだ印象を記録しながら、自分の好みを少しずつ理解できる仕組みに関心を持ちました。

### 10-3.キャッチアップ不足の懸念

キャッチアップ不足の段階では以下の懸念点があります。
1. **完成までのハードルが高い**
使用する言語やフレームワークが異なるため、キャッチアップにかかる時間は個人差があるものの、一定の時間を要することが予想されます。
2. **Issueの見積もりが難しい**
「どこまで理解していて、何が分からないのか」「調査にかかる時間」などが不明瞭な場合、開発期間内に完了することが難しくなってしまいます。

例えば、最近注目されているReactやNextについて。
Reactについては**本リリース後に組み込む方向で検討**するのが良いかと考えています。
その後、React単体のキャッチアップを行い、アプリに組み込む技術的な検証をした上で、組み込むかどうかを判断する形が理想かもしれません。
